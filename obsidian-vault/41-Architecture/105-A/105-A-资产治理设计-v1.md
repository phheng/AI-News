# 105-A 资产治理设计 v1

## 1) 资产命名规范 v1

## 1.1 通用格式
`<domain>-<capability>-<type>-v<major>`

示例：
- `amazon-ads-optimizer-skill-v1`
- `amazon-spapi-ingest-plugin-v1`
- `amazon-news-sync-trigger-v1`
- `discord-ops-channel-v1`
- `amazon-ops-playbook-experience-v1`

## 1.2 命名规则
- 全小写，使用 `-` 连接
- 禁止空格、中文标点、emoji
- `domain` 固定业务域（amazon/ecom/ops）
- `type` 必须显式标注（skill/plugin/trigger/channel/experience）

## 1.3 版本规则
- 大版本：`v1 -> v2`（不兼容变更）
- 小版本：文档内 `version: v1.1`（兼容优化）
- 市场发布保持语义化版本（如 1.0.0）并映射到内部 v1

## 2) 资产审核门禁 v1

## 2.1 风险分级
- 低风险：Skill / Experience
- 中风险：Trigger
- 高风险：Plugin / Channel

## 2.2 发布门禁

### Skill / Experience（快速通道）
- 必须项：元数据完整、示例可运行、来源链接可追溯
- 审核：1人审核即可

### Trigger（标准通道）
- 必须项：触发频率上限、失败重试策略、去重策略
- 审核：1人审核 + 1次灰度验证

### Plugin / Channel（严格通道）
- 必须项：权限清单、外部调用清单、错误与回滚机制
- 审核：2人审核（技术 + 安全）
- 上线：必须灰度（10%流量）

## 2.3 禁止发布条件
- 无来源可追溯
- 无回滚方案
- 自动执行高风险动作且无审批

## 3) 资产指标面板 v1

## 3.1 使用指标
- `installs_total`
- `active_tenants_7d`
- `runs_total_7d`

## 3.2 价值指标
- `adoption_rate`（建议采纳率）
- `outcome_lift`（业务指标改善）
- `retention_30d`

## 3.3 成本指标（重点）
- `token_in`
- `token_out`
- `token_total`
- `cost_per_run_usd`
- `cost_per_tenant_month_usd`
- `token_cost_ratio`（token成本/月费）

## 3.4 质量与风险指标
- `success_rate`
- `error_rate`
- `p95_duration_ms`
- `incident_count`
- `rollback_count`

## 3.5 看板节奏
- 日看板：成功率/错误率/token_total
- 周看板：采纳率/业务改善/活跃租户
- 月看板：成本占比/留存/转付费
