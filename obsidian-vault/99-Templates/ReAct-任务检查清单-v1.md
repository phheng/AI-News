# ReAct 任务检查清单（v1）

> 用法：每次任务执行前/后各过一遍，降低“执行了但没落地”的错误。

## 1) Reason（执行前）
- 任务编号：
- 任务目标（一句话）：
- 本次最小交付物：
- 输出路径（必须明确到文件夹）：
- 风险级别：低 / 中 / 高

## 2) Act（执行中）
- 仅执行本次最小必要动作
- 高风险动作是否先 dry-run：是 / 否
- 是否记录关键命令与参数：是 / 否

## 3) Observe（执行后）
- 状态：成功 / 失败 / 部分成功
- 证据1（日志/状态码）：
- 证据2（文件是否生成）：
- 证据3（是否推送成功）：

## 4) Reflect（复盘）
- 本次主要问题：
- 根因判断：
- 修正动作（下次默认规则）：
- 是否更新模板/流程：是 / 否
